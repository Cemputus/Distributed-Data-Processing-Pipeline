PART A:  PROJECT DESCRIPTION Distributed Data Processing Pipeline This project is intended to leverage distributed computing to process large datasets. Using the skill learned, demonstrate the application of MapReduce model, load balancing, cluster, Virtualisation management etc. The project should be structured as follows: a. Input Stage: In this stage, data should be stored in a distributed system (e.g., HDFS or AWS S3). b. Processing Stage: It should be hosted on a server like  Apache Spark or Hadoop cluster in the cloud to process data in parallel. c. Result Store: The aggregated data loaded into a data warehouse (e.g., Amazon Redshift, Google BigQuery, Snowflake, and Azure Synapse Analytics).  The completed project should be documented with: § System Architecture Diagram showing interactions between front-end, load balancers, servers, and storage. § Scalability Analysis showing how the system handles increasing loads. § Fault Tolerance Mechanisms showing steps taken if a node or service fails. § Security Configuration showing the IAM policies and network security    PART B: Project-based assessment guidelines  S/N Milestone Description Maximum Marks 1 MILESTONE ONE(Requirements specification- Planning) Develop system specifications to specify: • The system functional (features) and non functional (quality attributes like security, usability, and reliability) requirements. • Defines what the system must do and its performance constraints, producing a Software Requirements Specification (SRS) document on project goals, scope, and validation criteria.
